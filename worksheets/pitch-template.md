# Pitch Template — ใบงานครั้งที่ 9–10

> โครงการนำเสนอ 5–6 นาทีสำหรับ Demo Day — ทุกคนในทีมต้องพูดอย่างน้อย 1 ส่วน

## โครงสร้าง

### 1. ปัญหา (1 นาที) — ผู้พูด: ______

- ปัญหาคืออะไร เกิดที่ไหน บ่อยแค่ไหน
- ใครคือผู้ใช้/ผู้เดือดร้อน
- 💡 เปิดด้วยเรื่องจริงหรือคำถามที่ทำให้คนฟัง "อ๋อ เคยเจอ!"

### 2. ระบบทำงานอย่างไร (1 นาที) — ผู้พูด: ______

- โชว์ system diagram
- อธิบาย sense → think → act ของระบบ
- บอกว่า threshold มาจากไหน (ข้อมูลจริง!)

### 3. Demo สด (2 นาที) — ผู้สาธิต: ______

- สร้างเหตุการณ์จริงให้ระบบตอบสนองต่อหน้าทุกคน
- เล่าไปทำไป: "ตอนนี้ค่าอยู่ที่... พอผมทำแบบนี้... ระบบจะ..."
- ⚠️ เตรียมแผนสำรอง: วิดีโอ demo ที่อัดไว้ / อธิบายจากกราฟ

### 4. ข้อมูล + ผลทดสอบ (1 นาที) — ผู้พูด: ______

- โชว์กราฟจาก data.csv 1 รูป + ชี้ว่า threshold อยู่ตรงไหน
- สรุป testing: ทดสอบกี่ case, Pass กี่ case, bug ที่เจอและแก้แล้ว

### 5. ข้อจำกัด + อนาคต (1 นาที) — ผู้พูด: ______

- อะไรที่ระบบ **ยังทำไม่ได้** (พูดตรง ๆ ได้คะแนนมากกว่าอ้างว่าสมบูรณ์)
- ถ้ามีเวลา/งบต่อ จะพัฒนาอะไร

## Checklist ก่อนขึ้นเวที

- [ ] ซ้อมจับเวลาแล้ว ≤ 6 นาที
- [ ] `main.py` รันเองเมื่อเสียบไฟ (ไม่ต้องพึ่ง Thonny)
- [ ] มี power bank/สายสำรอง
- [ ] มีแผนสำรองถ้า demo พัง
- [ ] ทุกคนรู้ว่าตัวเองพูดส่วนไหน

## คำถามที่กรรมการชอบถาม (เตรียมคำตอบไว้)

1. ทำไมเลือก threshold ค่านี้
2. ถ้า sensor เสียกลางทาง ระบบจะเป็นยังไง
3. ระบบนี้ต่างจากของที่มีขายอยู่แล้วยังไง
4. ถ้าจะติดตั้งใช้จริงในโรงเรียน ต้องเพิ่มอะไร
