# Syllabus — Smart Sensor AIoT Engineering

## ข้อมูลทั่วไป

| หัวข้อ | รายละเอียด |
|---|---|
| ชื่อวิชา | Smart Sensor AIoT Engineering |
| กลุ่มเป้าหมาย | นักเรียน ม.5–ม.6 จำนวน 30 คน |
| รูปแบบ | Project-based Learning, 10 ทีม × 3 คน |
| เวลาเรียน | 10 ครั้ง × 2 ชั่วโมง = 20 ชั่วโมง |
| เครื่องมือ | ESP32, MicroPython, Thonny, GitHub |

## ปรัชญาของวิชา

วิชานี้ไม่ใช่ workshop ต่อวงจรให้ไฟติด แต่เป็นวิชา **วิศวกรรม** ที่เด็กต้องผ่านวงจรเต็ม:

```text
ปัญหาจริง → ออกแบบระบบ → สร้าง → เก็บข้อมูล → วิเคราะห์ → ปรับปรุง → ทดสอบ → นำเสนอ
```

## ผลการเรียนรู้ที่คาดหวัง (Learning Outcomes)

1. **Coding** — เขียน MicroPython ควบคุม GPIO, อ่าน sensor, เชื่อมต่อ Wi-Fi
2. **Hardware** — ต่อวงจรบน breadboard อย่างถูกต้องและปลอดภัย
3. **Data** — เก็บ บันทึก แสดงผล และตีความข้อมูลจาก sensor จริง
4. **System Thinking** — ออกแบบระบบ input → decision → output
5. **Testing** — ออกแบบ test case, บันทึกผล, วิเคราะห์ข้อผิดพลาด
6. **Communication** — บันทึกงานลง GitHub และนำเสนอแบบ pitch

## เกณฑ์การให้คะแนน (100 คะแนน)

| ส่วน | คะแนน | รายละเอียด |
|---|---|---|
| งานรายครั้ง (Mini Challenge + log) | 30 | ครั้งละ 3–4 คะแนน × 9 ครั้ง |
| GitHub Team Portfolio | 15 | ความสม่ำเสมอและคุณภาพของ README/log |
| Final Project — ตัวระบบ | 25 | ทำงานได้จริง, ใช้ sensor + logic + output ครบ |
| Testing & Data | 15 | testing table ≥ 10 case, data.csv, กราฟ, insight |
| Demo Day Presentation | 15 | ความชัดเจน, การตอบคำถาม, การแบ่งบทบาทในทีม |

### Rubric ของ Final Project (25 คะแนน)

| ระดับ | คะแนน | เกณฑ์ |
|---|---|---|
| Excellent | 21–25 | ระบบทำงานครบวงจร มี threshold จากข้อมูลจริง มี dashboard หรือการแจ้งเตือนหลายระดับ |
| Good | 16–20 | ระบบทำงานได้ มี sensor + alert ครบ แต่ threshold ตั้งแบบคาดเดา |
| Fair | 11–15 | อ่าน sensor และมี output ได้ แต่ logic ยังไม่สมบูรณ์ |
| Needs work | ≤10 | ระบบยังไม่ทำงานหรือไม่เสร็จ |

## เงื่อนไขการผ่านขั้นต่ำของทุกทีม

- [ ] อ่านค่า sensor ได้อย่างน้อย 1 ตัว
- [ ] มี logic ตัดสินใจอัตโนมัติอย่างน้อย 1 เงื่อนไข
- [ ] มี output แจ้งเตือน (LED/buzzer/web)
- [ ] มี `data.csv` และกราฟอย่างน้อย 1 รูป
- [ ] มี testing table อย่างน้อย 10 case
- [ ] มี README ทีมครบทุกสัปดาห์
- [ ] นำเสนอใน Demo Day

## นโยบายความปลอดภัย

- ใช้ไฟจาก USB 5V เท่านั้น ห้ามใช้แหล่งจ่ายไฟภายนอกโดยไม่ได้รับอนุญาต
- ถอด USB ก่อนแก้วงจรทุกครั้ง
- ตรวจวงจรกับเพื่อนในทีม (peer-check) ก่อนเสียบไฟ
