# Week 1: Introduction to Smart Sensor AIoT STEM Engineering

## หัวข้อประจำสัปดาห์

**เปิดโลกวิศวกรรมระบบอัจฉริยะ: จากปัญหาจริง สู่ Sensor, Data, AIoT และ STEM**

---

## เป้าหมายการเรียนรู้

เมื่อจบกิจกรรมครั้งนี้ นักเรียนจะสามารถ

1. อธิบายได้ว่า AIoT คืออะไร และเกี่ยวข้องกับวิศวกรรมอย่างไร
2. เข้าใจแนวคิด STEM ผ่านปัญหาใกล้ตัวในโรงเรียนหรือชุมชน
3. แยกองค์ประกอบของระบบอัจฉริยะได้เป็น Input → Process → Output
4. ตั้งทีมและเลือกปัญหาเบื้องต้นสำหรับทำ project
5. เขียน Problem Canvas ฉบับแรกของทีมได้

## ภาพรวมของ Track

ใน track นี้ นักเรียนจะได้สร้างต้นแบบระบบอัจฉริยะด้วย

* ESP32
* MicroPython
* Sensor
* Data
* Rule-based Intelligence / AI เบื้องต้น
* GitHub
* การออกแบบและทดสอบแบบวิศวกรรม

แนวคิดหลักของระบบคือ

```text
Sensor → Data → Decision → Output
```

ตัวอย่างเช่น

```text
วัดอุณหภูมิ → วิเคราะห์ข้อมูล → ตัดสินว่าร้อนเกินไป → แจ้งเตือนด้วย LED/Buzzer/Dashboard
```

---

### ช่วงที่ 2: AIoT System Thinking

เวลา: 25 นาที

ให้นักเรียนวิเคราะห์ระบบตัวอย่างด้วยโครงสร้าง

```text
Problem → User → Sensor/Input → Data → Decision → Output → Impact
```

ตัวอย่าง

| หัวข้อ       | ตัวอย่าง                                |
| ------------ | --------------------------------------- |
| Problem      | ห้องเรียนร้อนเกินไป                     |
| User         | นักเรียนและครู                          |
| Sensor/Input | DHT22 วัดอุณหภูมิและความชื้น            |
| Data         | ค่าอุณหภูมิในแต่ละช่วงเวลา              |
| Decision     | ถ้าอุณหภูมิสูงกว่า 30°C ให้แจ้งเตือน    |
| Output       | LED/Buzzer/Dashboard                    |
| Impact       | ช่วยให้ห้องเรียนเหมาะกับการเรียนมากขึ้น |

### ช่วงที่ 3: ตั้งทีม

เวลา: 15 นาที

จัดกลุ่มนักเรียนเป็นทีมละ 3 คน

บทบาทในทีม

| บทบาท             | หน้าที่                                |
| ----------------- | -------------------------------------- |
| Hardware Lead     | ต่อวงจร ดูแล sensor และอุปกรณ์         |
| Code/Data Lead    | เขียน MicroPython เก็บข้อมูล ทำกราฟ    |
| System/Pitch Lead | วาด system diagram ทำ README และนำเสนอ |

หมายเหตุ: บทบาทสามารถหมุนเวียนกันได้ ทุกคนควรได้ทดลองทั้ง hardware, coding และ presentation

### ช่วงที่ 4: เลือกปัญหา

2. Noise Alert System
3. Smart Plant Care
4. Smart Parking Mini
5. Safety Distance Alert
6. Light Energy Detective
7. Vibration Fault Detector
8. Mini Weather Station

เงื่อนไขของปัญหาที่ดี

* มีผู้ใช้ชัดเจน
* สามารถวัดข้อมูลได้
* ใช้ sensor ได้
* มีแนวทางสร้างต้นแบบภายใน 10 ครั้ง

### ช่วงที่ 6: สรุปท้ายคาบ

เวลา: 10 นาที

แต่ละทีมพูดสั้น ๆ ทีมละไม่เกิน 1 นาที

ตอบ 3 คำถาม

1. ทีมเลือกปัญหาอะไร
2. จะวัดข้อมูลอะไร
3. ระบบจะแจ้งเตือนหรือแสดงผลอย่างไร

---

## งานที่ต้องส่งท้ายคาบ

แต่ละทีมต้องส่ง

1. ชื่อทีม
2. รายชื่อสมาชิก
3. ปัญหาที่เลือก
4. STEM Problem Canvas ฉบับแรก
5. แนวคิดระบบแบบสั้น

ตัวอย่างการสรุประบบ

```text
ทีมของเราจะสร้างระบบ Smart Classroom Monitor 
โดยใช้ sensor วัดอุณหภูมิ แสง และเสียงในห้องเรียน 
จากนั้นนำข้อมูลมาตั้งเงื่อนไขว่า ห้องเรียนอยู่ในสภาพเหมาะสมหรือไม่ 
และแจ้งเตือนผ่าน LED หรือ Dashboard
```
